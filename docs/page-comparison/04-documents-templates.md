# Documents + Templates — Legacy vs New App

## Summary
Documents List: ~30 legacy items reviewed — roughly 3 Match, 8 Partial, 19 Missing. Document Detail Panel (legacy has no dedicated page; compared against Edit/Upload modals + per-file actions): ~28 legacy items reviewed — roughly 5 Match, 6 Partial, 16 Missing (plus 1 New page-shell addition). Templates: ~16 legacy items reviewed — roughly 9 Match, 3 Partial, 4 Missing.

## Document Detail Panel

*Legacy has no dedicated "document detail" page/route — compared here against the legacy Edit Document Request modal, Upload/Share Document modal, and per-file row actions, which together provide the equivalent functionality from the Documents list.*

| Legacy Item | Description | New App Status | New App Location | Notes |
|---|---|---|---|---|
| Dedicated per-document detail page | N/A — legacy handles everything via modals launched from the list row; there is no standalone URL/page per document. | New | `DocumentDetailPage.tsx` (`/documents/:requestId`) | A structural addition in the new app: every document request now has its own page with a breadcrumb back-link, not present in legacy at all. |
| "Client" field | Edit modal shows the client name as a read-only input (no navigation link). | Partial | `DocumentDetailPage.tsx` | New app shows the client name as a clickable link to the Client Detail page — an improvement, but not a literal legacy equivalent. |
